## 次のステップ — 型紙プレビューシステム

### フェーズ 1：平面プレビュー（2D）

ゲーム内キャラクター・アイテムの型紙（ベース画像）の上に、作成したドット画像を重ねて位置・サイズを調整するビュー。

#### 必要な機能

- **型紙レイヤー**
  - 型紙画像（PNG/SVG）を背景として読み込む
  - 透過対応（アルファチャンネルをそのまま表示）

- **ドットレイヤー**
  - 現在の Pixel Indexer の出力をそのままオーバーレイ
  - ドラッグで位置調整（x, y）
  - スライダーまたは数値入力でスケール調整
  - 回転（0 / 90 / 180 / 270 度、または自由角度）

- **表示制御**
  - 各レイヤーの表示 / 非表示トグル
  - 型紙の不透明度スライダー（重ねたときの視認性調整）
  - グリッド線のオン / オフ（位置合わせ補助）

- **出力**
  - 合成結果を PNG で保存（`p5.saveCanvas()` または Canvas API の `drawImage` 合成）
  - レイヤー設定（位置・スケール）を JSON でエクスポート / インポート（再現性のため）

#### 技術的な実装イメージ

```
p5.js の draw() 内で:
1. p.background(bgColor)
2. p.image(templateImg, 0, 0, w, h)          // 型紙レイヤー
3. p.image(dotCanvas,   offsetX, offsetY, dotW, dotH)  // ドットレイヤー
```

型紙は `Assets & Docs` タブから「→ Template」ボタンで送れるようにする（`sendToPixel` と同じ仕組み）。

### フェーズ 2：3D プレビュー

平面で確認した後、3D モデルにテクスチャとして貼り付けてイメージを確認する。

#### 技術選定候補

| ライブラリ | 特徴 | 向いているケース |
|-----------|------|----------------|
| **Three.js** | 最も実績あり、CDN で即使用可 | 汎用的な3Dプレビュー全般 |
| **Babylon.js** | Three.js より高機能、物理演算あり | ゲームエンジンに近い表現が必要な場合 |
| **model-viewer** | Google製、GLBファイルをタグ1行で表示 | 既存の3Dモデルファイルがある場合に最速 |

#### 最小構成のアプローチ（Three.js）

```
1. 単純な形状（Box / Plane / Cylinder など）を Three.js で表示
2. Pixel Indexer の p5 キャンバスを .toDataURL() で取得
3. Three.js の TextureLoader でそのまま貼り付け
4. OrbitControls でマウスドラッグ回転・ズーム
```

p5 キャンバスの内容をリアルタイムで Three.js テクスチャに反映することも可能（`texture.needsUpdate = true`）。

#### 想定する型紙の形状

- **平面（Plane）** — 旗・カード・シール類
- **直方体（Box）** — 箱型アイテム・建物
- **円柱（Cylinder）** — 缶・柱・武器の柄
- **カスタムメッシュ（GLB/OBJ）** — キャラクター本体など

## データ構造の設計メモ

型紙とドットの対応関係を管理するために、将来的に以下の JSON 構造を持つと再現性が高い。

```json
{
  "template": {
    "url": "assets/templates/base_body.png",
    "width": 64,
    "height": 64
  },
  "dot": {
    "gridSize": 8,
    "quantizeStep": 16,
    "swapMap": { "#000000": "#1a1a2e" },
    "cropRect": { "x": 10, "y": 5, "w": 44, "h": 54 }
  },
  "layout": {
    "offsetX": 0,
    "offsetY": 0,
    "scale": 1.0,
    "rotation": 0
  }
}
```

## 懸念点・注意事項

- **CORS**: `raw.githubusercontent.com` の画像を Three.js テクスチャに使う場合、`crossOrigin = 'anonymous'` の設定が必要
- **p5 と Three.js の共存**: 同一ページに両方置く場合、キャンバス要素を明示的に分けて管理する（p5 は `instance mode` で使用中なので競合しにくい）
- **モバイル対応**: タッチイベントでのドラッグ・ピンチズームは別途対応が必要
- **パフォーマンス**: 3D テクスチャのリアルタイム更新は重くなりやすいので、「適用」ボタンを押したときだけ更新する設計が無難
